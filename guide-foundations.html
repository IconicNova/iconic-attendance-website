4. Buffer Management & Stream Handling

4.1 - cin >> vs getline()

Critical Difference
ğŸ”¥ The Problem:
cin >> leaves the newline character (\n) in the input buffer! 
// PROBLEM SCENARIO:
int choice;
cin >> choice;        // User types "3"
                     // cin reads "3", leaves "\n" in buffer

string name;
getline(cin, name);  // Immediately reads "\n" (empty line!)
                     // name becomes empty string ""
Solution: Use cin.ignore() to clear the buffer 
int choice;
cin >> choice;
cin.ignore(numeric_limits::max(), '\n');  // Clear buffer

string name;
getline(cin, name);  // Now reads actual input correctly

4.3 - Conditional Buffer Clearing

Multi-Period Mode Pattern

getline(cin, multiPeriodInput);  // Already clears buffer

// PATTERN 2: cin >> then getline (must ignore)

                         multiPeriodInput.find('-') != string::npos);

    cin.ignore(numeric_limits::max(), '\n');
    // Only ignore if we used cin >>, not if we used getline
}

5. String Processing & Parsing

5.1 - Date Parsing Pattern

DD-MM-YYYY Format
// INPUT: "27-11-2025"
int day, month, year;
char sep1, sep2;

stringstream ss(dateStr);
ss >> day >> sep1 >> month >> sep2 >> year;
//    27      -       11       -      2025

// VALIDATION:
if (sep1 != '-' || sep2 != '-') {
    // Invalid format (not using dashes)
    return false;
}

// POSITION VALIDATION:
if (dateStr.length() != 10 || dateStr[2] != '-' || dateStr[5] != '-') {
    // Not exactly 10 characters, or dashes in wrong positions
    return false;
}
How >> Operator Works:
Extracts tokens separated by whitespace OR until reaching non-matching type
ss >> day reads digits until '-'
ss >> sep1 reads the '-' character
ss >> month reads next digits
And so on...

5.2 - Time Parsing Pattern

HH:MM AM/PM Format

stringstream ss(timeSlot);

// Now can reformat with zero padding:

       << setfill('0') << setw(2) << minute << " " << period;
// "10:40 AM" (already padded) or "09:05 AM" (with padding)

7. Zeller's Congruence - Day of Week Algorithm

Complete Algorithm
Calculate day of week from any date (Gregorian calendar)
int getDayOfWeek(int day, int month, int year) {
    // STEP 1: Adjust for January/February
    if (month < 3) {
        month += 12;  // January = 13, February = 14
        year--;       // Treat as previous year
    }
    
    // STEP 2: Calculate components
    int k = year % 100;        // Year within century (0-99)
    int j = year / 100;        // Century number
    
    // STEP 3: Zeller's formula
    int h = (day + (13 * (month + 1)) / 5 + k + k/4 + j/4 - 2*j) % 7;
    
    // STEP 4: Convert to 0=Sunday, 1=Monday, etc.
    int dayOfWeek = (h + 5) % 7 + 1;  // Adjust to 1=Sunday, 2=Monday, ...
    
    return dayOfWeek;
}
Why January/February Special?
Gregorian calendar has leap days at end of February
Treating Jan/Feb as months 13/14 of previous year makes leap year math easier

Example:
January 15, 2025 â†’ Becomes month 13, year 2024
Step-by-Step Example: November 27, 2025 
Given: day=27, month=11, year=2025

// month >= 3, no adjustment needed

k = 2025 % 100 = 25                    (year in century)
j = 2025 / 100 = 20                    (century)

h = (27 + (13 * 12) / 5 + 25 + 25/4 + 20/4 - 2*20) % 7
h = (27 + 156/5 + 25 + 6 + 5 - 40) % 7
h = (27 + 31 + 25 + 6 + 5 - 40) % 7
h = 54 % 7
h = 5

dayOfWeek = (5 + 5) % 7 + 1 = 10 % 7 + 1 = 3 + 1 = 4
// 4 = Wednesday âœ“
Historical Context:
Formula developed by Christian Zeller (1822-1899)
Accounts for Gregorian calendar reform (1582)
-2*j term adjusts for century leap year rule
